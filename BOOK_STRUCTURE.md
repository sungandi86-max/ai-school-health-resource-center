# BOOK_STRUCTURE

## 문서 목적

이 문서는 『보건교사를 위한 AI 업무 자동화』 최종 조판본의 전체 구성을 페이지 단위로 기록합니다.

새로운 목차나 권장 분량을 제안하는 문서가 아닙니다. 최종 출간본에 실제로 배치된 표지, 차례, 8개 PART, 22개 Chapter, PART 마무리와 에필로그의 순서와 연결 관계를 정리합니다.

## 확인 범위

- 최종 조판본 전체 211쪽
- 차례 2–3쪽
- PART 표지와 PART 도입

## 책의 진행 방향

출간본의 8개 PART는 다음 순서로 이동한다.

```text
바쁜 하루 관찰
→ 반복 장면 발견
→ 사람·AI·정보의 경계 설정
→ 첫 개선 과제 선택
→ 화면부터 만든 실패 기록
→ 업무를 작은 행동으로 재설계
→ 행과 열의 구조 만들기
→ 안내에서 완료까지 Workflow 연결
→ 반복 한 부분에 AI 연결
→ 실제 학교에서 사용·수정·공유
→ 운영 기준 남기기
```

이 순서는 기술 기능의 난이도를 높이는 순서가 아니라, 한 프로젝트가 관찰에서 운영으로 이어지는 순서로 확인된다.

### PART 1. 왜 보건실은 늘 바쁠까

| 항목 | 내용 |
|---|---|

PART 1은 문제를 해결하지 않는다. 바쁨을 개인의 속도 문제로 보던 상태에서 반복되는 행동을 관찰 가능한 기록으로 바꾸는 데서 끝난다.

| 항목 | 내용 |
|---|---|

PART 2는 무엇을 자동화할지보다 먼저 사람이 끝까지 판단할 일과 입력·공개하지 않을 정보를 정한다. 그 뒤 여러 반복 중 하나만 첫 프로젝트로 남긴다.

### PART 4. 화면을 닫고, 표부터 다시 만들다

## 결과물의 누적 구조

각 Chapter의 결과물은 독립된 활동지가 아니라 다음 Chapter의 입력으로 다시 사용된다.

```text
하루 한 장면 해부표
└─ 반복 장면 기록지
   └─ 사람·AI·정보 경계표
      └─ 첫 개선 과제 선정표
         └─ 첫 이상 징후 기록
            └─ 사용자 흐름 실패 지도
               └─ 재설계 기준 메모
                  └─ 업무 분해 초안
                     └─ 행동 기준표
                        └─ 업무 설계표
                           └─ 업무 단위 정의표
                              └─ 컬럼 설계표
                                 └─ 첫 업무 구조
                                    └─ Workflow 초안
                                       └─ 중복 제거표
                                          └─ 첫 Workflow 완성본
                                             └─ 첫 AI 요청문
                                                └─ AI 역할표
                                                   └─ AI Workflow 1.0
                                                      └─ 운영 기록표
                                                         └─ 운영 체크리스트
                                                            └─ 운영 선언문
```

실제 본문에서는 모든 결과물을 매번 하나의 파일에 병합한다고 표현하지 않는다. 대신 이전 PART 또는 Chapter에서 만든 표와 기록을 다시 펼치고, 옆에 놓고, 연결하거나 수정하는 방식이 반복된다.

- PART 1–7은 각 PART 안의 Chapter가 같은 DAY를 공유한다.
- PART 8은 Chapter마다 DAY가 60, 75, 90으로 증가한다.
- DAY는 Chapter 수와 일대일로 증가하지 않는다.
- 최종 출간본은 DAY 1에서 DAY 90까지의 프로젝트 진행을 표시한다.

## 난이도 구조

- 실제 사용 범위는 ★☆☆☆☆부터 ★★★☆☆까지다.
- ★★★★☆와 ★★★★★는 출간본에서 확인되지 않는다.
- Chapter 1만 ★☆☆☆☆다.
- Chapter 2–5, 8, 17, 22는 ★★☆☆☆다.
- 나머지 Chapter는 ★★★☆☆다.
- 뒤로 갈수록 계속 높아지는 단순 상승 구조는 아니다.
- 마지막 Chapter 22는 전체 프로젝트의 결론이지만 난이도는 ★★☆☆☆로 표시된다.

## 시간 구조

- Chapter별 예상 시간은 약 20–45분 범위다.

- PART 7: 174–175쪽에 `PART 7을 마치며`
- PART 8: 201–202쪽에 `PART 8을 마치며`와 `에필로그 Preview`

### 기능

PART 마무리는 다음 내용을 수행한다.

1. 해당 PART에서 만든 결과를 되짚는다.
2. 아직 하지 않은 것 또는 남은 질문을 밝힌다.
3. 다음 PART에서 사용할 이전 결과물을 남긴다.
4. 다음 단계의 질문이나 행동으로 연결한다.

마무리의 위치와 길이는 PART마다 같지 않다. 별도 고정 페이지 수는 확인되지 않는다.

## 프로젝트 진행률

### Chapter 진행률

프로젝트 진행 카드는 PART 안의 현재 위치를 다음처럼 표시한다.

- 첫 Chapter: `PART 진행 시작 · 0 / n`
- 중간 Chapter: `PART 진행 진행 중 · x / n`
- Chapter 결과 영역: `PART n 진행률 · x / n 완료`

### 전체 진행률

전체 프로젝트 진행률은 일부 PART 마지막 결과에서 확인된다.

- PART 5: `전체 프로젝트 진행률 · 5 / 8 완료`
- PART 6: `전체 프로젝트 진행률 · 6 / 8 완료`
- PART 8: `전체 프로젝트 진행률 · 8 / 8 완료`

모든 PART 끝에 전체 진행률이 표시되는 것은 아니다. 따라서 전체 진행률 표시는 각 PART의 필수 고정 요소로 확정하지 않는다.

## 구조에서 확인되는 핵심 연결 원리

### 1. 해결보다 관찰이 먼저다

PART 1은 해결책을 만들지 않고 반복 장면을 기록한다.

### 2. AI보다 경계가 먼저다

AI를 실제로 사용하는 PART 7보다 앞선 PART 2에서 사람·AI·정보 경계를 먼저 만든다.

### 3. 실패를 삭제하지 않는다

PART 3 전체를 첫 화면의 실패 기록에 사용하고, 이 실패가 PART 4의 재설계 입력이 된다.

### 4. 화면보다 업무 구조가 먼저다

PART 4–6에서 행동, 기준, 행, 열, Workflow를 먼저 만든다.

### 5. AI는 완성된 Workflow의 일부에 들어간다

AI는 Chapter 17에서 처음 등장하지 않는다. 앞에서 경계를 설명할 때부터 언급되지만, 실제 요청문과 역할 연결은 첫 Workflow가 완성된 뒤인 PART 7에서 수행한다.

### 6. 완성 뒤에 운영이 남는다

PART 8은 공유, 사용자 관찰, 수정, 공동 운영 기준과 선언문을 다룬다.

### 7. 마지막 결과가 처음 문제를 다시 해석한다

Chapter 22와 에필로그는 Chapter 1의 `내가 느려서 바쁘다`는 해석을 반복과 운영의 문제로 다시 읽는다.

## 확인된 예외

- PART별 Chapter 수는 동일하지 않다. PART 1–2는 2개, PART 3–8은 3개다.
- PART 1에는 별도 `PART 1을 마치며` 제목이 없다.
- PART 5–8에만 PART 표지 다음 별도 도입 페이지가 있다.

- 모든 PART 끝에 전체 프로젝트 진행률이 표시되지는 않는다.
- 에필로그는 Chapter 공통 컴포넌트를 사용하지 않는다.

## 이 문서에서 확정하지 않는 항목

다음 내용은 최종 출간본에서 새로운 책의 고정 규칙으로 확인되지 않는다.

- 모든 책이 반드시 8개 PART여야 한다는 기준
- 모든 책이 반드시 22개 Chapter여야 한다는 기준
- 모든 프로젝트가 반드시 DAY 90까지 진행되어야 한다는 기준
- PART별 고정 페이지 수
- Chapter별 고정 페이지 수
- 모든 주제에서 같은 결과물 이름을 사용해야 한다는 기준
- 모든 새 책의 난이도를 3단계까지만 사용해야 한다는 기준

이 문서는 출간본의 구조를 기록한다. 반복 가능한 PART와 Chapter 구성 요소는 `PART_TEMPLATE.md`와 `CHAPTER_TEMPLATE.md`에서 별도로 정리한다.
