# 전자책 제작 Prompt Pack

버전: v1.1
업데이트: 2026-07-14

전자책 기획, 원고 검토, 실습 자료 점검에 사용하는 프롬프트 모음입니다. 실제 학생,
교직원, 학교 내부 정보를 입력하지 마세요.

## 1. 목차 구조 점검

```text
다음 전자책 기획 정보를 바탕으로 목차 구조를 점검해주세요.

독자: {{독자}}
책의 목표: {{목표}}
예상 분량: {{분량}}
현재 목차: {{목차}}

점검 기준:
- 각 장이 독자의 실제 업무 흐름과 연결되는지
- 앞 장의 개념이 뒤 장의 실습에 필요한 순서인지
- 설명이 중복되는 장이 있는지
- 실습 자료나 체크리스트가 필요한 장이 어디인지

결과는 '장 / 역할 / 보완할 내용 / 필요한 실습 자료' 표로 작성해주세요.
```

## 2. 원고 검토

```text
아래 원고를 전자책 독자의 관점에서 검토해주세요.

원고: {{원고}}

검토 항목:
1. 처음 읽는 독자가 이해하기 어려운 용어
2. 단계가 생략된 설명
3. 예시가 필요한 부분
4. 개인정보 보호 안내가 필요한 부분
5. 최신 공식 자료 확인이 필요한 문장

원문을 임의로 확정하지 말고 수정 제안과 담당자 확인 항목을 분리해주세요.
```

## 3. 장별 실습 자료 점검

```text
다음 장의 학습 목표와 본문을 기준으로 필요한 추가 자료를 제안해주세요.

장 제목: {{장제목}}
학습 목표: {{학습목표}}
본문 요약: {{본문요약}}

자료 유형은 Prompt, Template, Workflow, Checklist, Example 중에서 선택하고,
각 자료가 본문의 어느 단계에서 사용되는지 설명해주세요.
실제 개인정보가 필요한 예시는 제안하지 마세요.
```

## 4. 최종 개인정보 검토

```text
다음 전자책 원고와 실습 예시에서 개인정보 또는 민감정보 위험을 점검해주세요.

검토 대상: {{검토대상}}

결과 형식:
- 발견 위치
- 위험한 정보 유형
- 익명화 또는 일반화 방법
- 담당자가 확인할 사항

실제 인물이나 학교를 추정하지 말고 텍스트에 명시된 정보만 검토해주세요.
```
