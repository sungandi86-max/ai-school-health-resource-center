# Chapter 07. 문제는 화면이 아니라 순서였습니다

## 재설계 기준 메모 프롬프트

아래 실패 기록에서 반복해서 빠진 질문을 찾아 재설계 기준으로 바꿔주세요.

[실패 기록]

다섯 기준으로 정리합니다.
1. 사용자와 목적
2. 한 번의 입력이 뜻하는 것
3. 서로 다른 업무를 구분하는 기준
4. 공개 정보와 내부 정보의 경계
5. 실제 사용자의 확인 순서

구체적인 앱이나 코드 해결책은 제안하지 마세요.

---

## 사용 전 확인

- 실제 학생 이름, 학번, 질병명, 검사 결과, 상담 내용을 입력하지 않습니다.
- 더미 데이터로 먼저 시험합니다.
- 결과는 보건교사가 최종 확인합니다.
