# Codex 작업 원칙

## 기본 원칙

- 작업 전 `README.md`, `PROJECT.md`, `IA.md`, `DESIGN.md`, `CONTENT_MODEL.md`, `ROADMAP.md`를 확인합니다.
- 기존 기능과 문서의 의도를 보존합니다.
- 로그인, 데이터베이스, Supabase는 명시 요청 전 구현하지 않습니다.
- shadcn/ui, MDX, Fuse.js는 필요성이 확인된 다음 단계에서만 추가합니다.
- 새로운 대형 패키지는 도입 이유와 대안을 먼저 검토합니다.
- 한국어 UI를 기본으로 작성합니다.
- 실제 학생 및 교직원의 개인정보 예시는 만들지 않습니다.

## 코드 변경 전 확인할 문서

1. `README.md`: 프로젝트 방향과 운영자 의도
2. `PROJECT.md`: MVP 범위와 제외 범위
3. `IA.md`: 메뉴와 URL 구조
4. `DESIGN.md`: 색상, 타이포그래피, 컴포넌트 원칙
5. `CONTENT_MODEL.md`: 콘텐츠 데이터 구조
6. `ROADMAP.md`: 현재 단계와 다음 단계 범위

## 기능 보존 원칙

- 사용자가 요청하지 않은 기능을 임의로 추가하지 않습니다.
- 이미 정의된 메뉴와 사용자 흐름을 깨지 않습니다.
- 자료, Workflow, Project, Update Log의 연결 구조를 유지합니다.
- 개인정보 보호 안내를 약화하지 않습니다.

## 디자인 일관성 원칙

- 모바일 우선으로 설계합니다.
- 화이트 기반, 연한 블루 그레이 배경, 네이비 텍스트를 기본으로 합니다.
- 블루는 검색과 주요 탐색 액션에 사용합니다.
- 그린은 복사, 다운로드, 성공 상태에 사용합니다.
- 카드 반경은 8px 내외로 유지합니다.
- 과한 그림자, 과한 그라데이션, 장식용 일러스트를 사용하지 않습니다.
- 거대한 Hero 영역 대신 실제 업무 탐색을 우선합니다.

## 검증 명령

```bash
npm run lint
npm run build
git diff --check
```

UI를 변경한 경우에는 로컬 서버에서 모바일과 데스크톱 화면을 직접 확인합니다.

- 프로젝트 구조
- 문서별 핵심 내용
- 실행한 검증 명령
- 검증 결과
